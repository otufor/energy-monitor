# Interactive HTML

## 目的

建築図面のように固定的に見せるのではなく、ユーザーが観点を切り替えながら理解を深められる HTML を作る。

## 最低限の構成

- タイトルとプロジェクト概要
- 観点切替タブ
- レイヤーや責務を選べるメイン図
- 右側または下部の詳細ペイン
- 実ファイル対応表

## あると強い要素

- 現状案と改善案の切替
- レイヤーごとの課題表示
- 「この責務はここに移すと良い」の注釈
- ファイルパスをコピーできる UI

## 実装のコツ

- 依存ゼロの単一 HTML を優先
- CSS 変数でテーマを揃える
- JavaScript は状態切替を小さく保つ
- 出力先は `/tmp` または `docs/tmp`

## 禁止したいこと

- 汎用的すぎる図だけ出してコードベースとの対応がない
- 色やレイアウトだけ派手で情報密度が低い
- 1つの正解を押し付ける表現
