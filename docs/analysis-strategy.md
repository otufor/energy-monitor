# 多角的分析戦略

## 1. 目的

energy-monitor における分析の目的は、単に消費電力を可視化することではなく、家庭内で起きた電力ピークや高負荷イベントについて、

- いつ起きたか
- どの程度大きかったか
- 何が主因だった可能性が高いか
- 次にどう行動すべきか

を、納得感のある形で説明できるようにすることである。

そのため分析は、単一のグラフや単一指標ではなく、複数の視点を組み合わせて「原因仮説を絞り込む」構成にする。

## 2. 分析の基本方針

### 2.1 分析は 3 層で設計する

1. **観測**
   現在値、推移、ピーク、しきい値超過などを把握する。
2. **解釈**
   その変化が一時的なものか、習慣的なものか、異常かを見分ける。
3. **行動**
   再発防止や生活改善につながる示唆に変換する。

### 2.2 「原因断定」ではなく「原因仮説の優先順位付け」を目指す

家庭の電力データだけで家電を完全に特定するのは難しい。したがって、分析のゴールは厳密な断定ではなく、以下のように主因候補を整理することに置く。

- ベース負荷の上昇
- 短時間の急上昇
- 特定時間帯に繰り返す周期負荷
- 気温や生活イベントに連動する負荷
- 複数要因の重なり

### 2.3 日常利用と深掘り分析を分ける

- ダッシュボードでは、日々の確認と異常検知を重視する
- notebook / レポートでは、イベントの事後分析と比較分析を重視する

## 3. 多角的に見るための分析軸

### 3.1 時系列軸

最も基本となる軸。まずは「いつ・どれだけ使ったか」を押さえる。

- 直近 60 分の推移
- 1 日の推移
- 数日〜数週間の連続推移
- ピーク発生前後 30〜120 分の文脈
- 移動平均と瞬時値の乖離

見るべきポイント:

- 急峻な立ち上がりか、じわじわ増える型か
- ピークが単発か連続か
- 高負荷が何分継続したか

### 3.2 分布軸

平均だけでは実態を見誤るため、分位点や頻度分布で負荷の広がりを見る。

- 平均、中央値
- P95、P99
- 高負荷しきい値超過分数
- ワット数帯ごとの滞在時間

見るべきポイント:

- 普段の生活水準に対してピークがどれほど外れているか
- 「たまに高い」のか「全体に高い」のか

### 3.3 イベント軸

ピークそのものを 1 件ずつイベントとして扱う。

- 日次ピーク
- しきい値超過イベント
- 急上昇イベント
- 長時間高負荷イベント

イベントごとに最低限残したい情報:

- 発生時刻
- 最大電力
- 継続時間
- 発生前 15 分 / 30 分の平均負荷
- 収束までの時間

### 3.4 比較軸

単日の観測だけでは原因が見えにくいので、比較で特徴を浮かび上がらせる。

- 前日比
- 同曜日比
- 平日 / 休日比較
- 月初 / 月末比較
- 寒い日 / 暖かい日比較
- 在宅が多い日 / 少ない日比較

見るべきポイント:

- その日だけ特殊なのか
- 繰り返し起きる再発パターンか

### 3.5 文脈軸

「なぜそのタイミングで増えたのか」を理解するための補助軸。

候補:

- 外気温
- 天候
- 時間帯
- 曜日
- 起床、食事、入浴、就寝などの生活イベント
- 暖房利用の有無
- 給湯関連のタイミング

この軸は v1 時点では十分に自動取得できなくてもよい。まずは手入力メモや簡易タグでも十分価値がある。

### 3.6 変化点軸

電力の絶対値だけでなく、「どこで状態が切り替わったか」を見る。

- 前分差分
- 5 分平均との差
- 立ち上がり速度
- 急落の有無
- 状態遷移点の抽出

これにより、連続運転型の家電と、短時間に強く効く家電を見分けやすくなる。

## 4. まず追うべき分析質問

分析では、毎回ゼロから眺めるのではなく、同じ問いの順番で見ると再現性が上がる。

### 4.1 イベント理解の基本質問

1. いつピークが起きたか
2. 何分続いたか
3. 直前からどれだけ増えたか
4. その日の中で特異だったか
5. 過去にも似たイベントがあったか
6. 気温や生活イベントと重なっていたか
7. 再発防止のために避けるべき重なりは何か

### 4.2 日次理解の基本質問

1. その日の総使用量は多かったか
2. ピークが押し上げた日のか、終日高めだった日のか
3. 朝・昼・夜のどこに偏っていたか
4. 普段と違う生活パターンがあったか

## 5. 指標設計

### 5.1 v1 で必ず持つべき指標

- 平均電力
- 中央値
- 最大電力
- P95
- P99
- 日次消費電力量
- 高負荷しきい値超過分数
- ピーク発生時刻
- ピーク前後の平均負荷
- ピーク継続時間

### 5.2 次段階で追加したい指標

- 急上昇回数
- 連続高負荷イベント数
- 時間帯別平均負荷
- ベース負荷
- 曜日別平均
- 気温帯別平均負荷
- 再発イベント類似度

## 6. 分析成果物の役割分担

### 6.1 ダッシュボード

役割は「日常の観測」と「異常の早期発見」。

向いているもの:

- 直近 60 分のグラフ
- 当日サマリー
- 高負荷アラート
- しきい値超過の可視化
- 前日比の簡易表示

### 6.2 日次レポート

役割は「昨日何が起きたか」の振り返り。

向いているもの:

- 日次総使用量
- ピーク時刻と前後推移
- 時間帯別の負荷傾向
- 前日 / 先週同曜日との比較
- 特記事項の自動コメント

### 6.3 notebook

役割は「原因追跡」と「仮説検証」。

向いているもの:

- 期間横断比較
- ピークイベント一覧
- 閾値変更シミュレーション
- 気温や生活メモとの突合
- オーバーフローや欠損を考慮した再計算

## 7. 現在の実装から見た強みと不足

### 7.1 すでに活かせる強み

- 毎分データがあるため、ピーク前後の追跡ができる
- `cum_raw` と `cum_kwh` を保持しており、再計算や差分検証に強い
- notebook で期間分析ができる
- ダッシュボードとバッチレポートの両輪を持てる

### 7.2 多角分析のために不足しているもの

- 気温・天候などの外部文脈データ
- 生活イベントの手入力メモ
- イベント単位でのラベリング
- 平日 / 休日や曜日比較の定型ビュー
- 長期間 API 利用時の集約、ページネーション

## 8. 実装優先順位

### Phase 1: すぐ始める

既存データだけで、原因追跡の精度を一段上げる。

- ピークイベント一覧を notebook に追加する
- 高負荷イベントごとの継続時間を算出する
- 前後 30 分比較を定型化する
- 前日比、同曜日比を追加する
- ベース負荷とピーク負荷を分けて見る

### Phase 2: 比較可能性を強化する

イベントを単発で終わらせず、再発パターンを見つけやすくする。

- 曜日別、時間帯別の集計
- 平日 / 休日比較
- 週次サマリー
- イベント類型の分類
  - 短時間急上昇型
  - 長時間高負荷型
  - 周期反復型

### Phase 3: 文脈を取り込む

「なぜ」をより説明可能にする。

- 気温データの取得と突合
- 手入力メモの保存
- 生活イベントタグ付け
- 気温帯別のピーク発生傾向分析

### Phase 4: 示唆を返す

観測結果を、具体的な行動につなげる。

- 「この時間帯の併用が危険」というコメント生成
- 再発頻度の高いパターン抽出
- しきい値見直し提案
- 節電施策の前後比較

## 9. notebook で最初に追加したい分析メニュー

優先度が高い順に、以下を実装すると戦略との整合が取りやすい。

1. **ピークイベント一覧**
   発生時刻、最大電力、継続分数、直前平均との差を一覧化する。
2. **ピーク前後比較ビュー**
   ピークの前後 30 分を切り出し、立ち上がり方と収束の仕方を見る。
3. **日別比較ビュー**
   日次総量、ピーク、P95、高負荷分数を並べて比較する。
4. **時間帯別ヒートマップ**
   何時台に負荷が偏るかを把握する。
5. **ベース負荷推定**
   深夜帯などから最低水準を推定し、異常な底上がりを検出する。

## 10. 意思決定ルール

分析機能を増やすときは、以下の順に価値を判定する。

1. ピークの原因理解に近づくか
2. 再発防止や生活改善に使えるか
3. notebook だけでなく日常導線にも落とせるか
4. 家庭利用として複雑にし過ぎないか

## 11. まとめ

energy-monitor の多角的分析戦略は、時系列・分布・イベント・比較・文脈・変化点の 6 軸を組み合わせて、電力ピークの原因仮説を絞り込むことにある。

最初にやるべきことは、既存の毎分データからイベント単位の見方を強化し、前後比較と日別比較を定型化すること。そのうえで、気温や生活メモなどの文脈を足していくことで、「高かった」から「なぜ高かったか」へ分析を進化させられる。
