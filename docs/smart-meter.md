## ECHONET Lite プロパティコード (EPC)

Nature Remo API のレスポンス中 `smart_meter.echonetlite_properties` に含まれる。

| EPC (hex) | EPC (dec) | 名称             | 単位 | 備考                            |
| --------- | --------- | ---------------- | ---- | ------------------------------- |
| `0xE7`    | 231       | 瞬時電力計測値   | W    | 現在の消費電力                  |
| `0xE8`    | 232       | 瞬時電流計測値   | A×10 | **10 倍値** → ÷10 して A に変換 |
| `0xE0`    | 224       | 正方向積算電力量 | —    | 買電の生カウンタ値              |
| `0xE3`    | 227       | 逆方向積算電力量 | —    | 売電の生カウンタ値（太陽光等）  |
| `0xD3`    | 211       | 係数             | —    | デフォルト 1（未設定時）        |
| `0xE1`    | 225       | 積算電力量単位   | —    | 単位コード表を参照              |
| `0xD7`    | 215       | 有効桁数         | —    | デフォルト 6（最大値 999,999）  |

> **重要**: `val` フィールドは**10進数文字列**で返される。必ず `parseInt(val, 10)` で数値に変換すること。
> 実 API レスポンス例: `epc=231 val='3086'` → 3086 W

## 積算電力量単位コード (EPC `0xE1`)

| コード (hex) | 係数 (kWh)       |
| ------------ | ---------------- |
| `0x00`       | 1                |
| `0x01`       | 0.1 ← 最も一般的 |
| `0x02`       | 0.01             |
| `0x03`       | 0.001            |
| `0x04`       | 0.0001           |
| `0x0A`       | 10               |
| `0x0B`       | 100              |
| `0x0C`       | 1000             |
| `0x0D`       | 10000            |

EPC が未設定の場合は `0x01`（0.1 kWh）をデフォルト値として使用する。

## 実積算電力量の計算式

```
実積算電力量 (kWh) = cum_raw × 係数 × unitFactor(単位コード)
```

### 計算例

| 値                        | 内容                           |
| ------------------------- | ------------------------------ |
| `cum_raw` = 5167          | EPC `0xE0` の生値              |
| 係数 = 1                  | EPC `0xD3`                     |
| 単位コード = `0x01` → 0.1 | EPC `0xE1`                     |
| **結果**                  | 5167 × 1 × 0.1 = **516.7 kWh** |

## 消費電力量の差分計算

ある期間の消費電力量は積算値の差分で求める。

```
消費電力量 = cum_kwh(終了時刻) - cum_kwh(開始時刻)
```

例（1 日分）:

```sql
SELECT ROUND(MAX(cum_kwh) - MIN(cum_kwh), 3) AS total_kwh
FROM power_log
WHERE ts LIKE '2025-01-01%'
```

## オーバーフロー処理

積算電力量カウンタは有効桁数（`0xD7`）で定まる最大値に達すると **0 にリセット**される。

- 6 桁の場合: 最大値 = 999,999（一般的な家庭用スマートメーター）
- 差分計算でリセットが跨ぐと**負の値**になるため検知が必要

### 実装方針

1. DB に保存する際に直前の `cum_raw` と比較する
2. 現在値 < 直前値 の場合はオーバーフローとみなし LINE 通知する
3. Jupyter での月次・日次解析時は差分が負になるレコードを除外する

```python
# Jupyter での差分計算例（オーバーフロー考慮）
df["delta_kwh"] = df["cum_kwh"].diff()
df = df[df["delta_kwh"] >= 0]  # オーバーフロー行を除外
daily_kwh = df["delta_kwh"].sum()
```

## アプライアンス type 値

スマートメーターを識別する `type` フィールドの正しい値は **`"EL_SMART_METER"`**。
`"EL"` ではないため注意。

```typescript
const sm = appliances.find((a) => a.type === "EL_SMART_METER");
```

## 瞬時電流の変換

EPC `0xE8` は **A×10 の16進数文字列**で返ってくる。

```typescript
const ampere = parseInt(prop.val, 16) / 10;
// 例: val = "37" (hex) → 0x37 = 55 → 5.5 A
```

## val の10進数変換まとめ

```typescript
// すべての EPC の val は10進数文字列
const value = parseInt(val, 10);

// 瞬時電力 (W)
const watts = parseInt(val, 10); // 例: "3086" → 3086 W

// 瞬時電流 (A×10) ※API に含まれない場合あり
const ampere = parseInt(val, 10) / 10; // 例: "55"  → 5.5 A

// 積算電力量（生カウンタ値）
const cum_raw = parseInt(val, 10); // 例: "837062" → 837062
```

## 実 API レスポンス例

```
epc=211 (0xD3) val='1'       # 係数 = 1
epc=215 (0xD7) val='6'       # 有効桁数 = 6桁（最大 999,999）
epc=224 (0xE0) val='837062'  # 積算電力量 → 83706.2 kWh
epc=225 (0xE1) val='1'       # 単位コード 1 → 0.1 kWh
epc=227 (0xE3) val='9'       # 逆方向積算 → 0.9 kWh（売電）
epc=231 (0xE7) val='3086'    # 瞬時電力 = 3086 W
```
